## 2. 不同城市对比测试

在 2026-05-21 18:20 左右调用 OpenWeatherMap 当前天气接口，对比不同城市的降雨强度：

| 城市 | 降雨强度 | 告警等级 | 天气描述 |
| --- | ---: | --- | --- |
| Beijing | 0.0 mm/h | 绿色 | 阴，多云 |
| Shanghai | 0.0 mm/h | 绿色 | 多云 |
| Guangzhou | 0.3 mm/h | 绿色 | 小雨 |

结论：

- 北京和上海当前无明显降雨，0.0 mm/h 是合理结果。
- 广州显示小雨，0.3 mm/h 属于很弱降雨，未达到 10 mm/h 的黄色告警阈值，绿色状态合理。

## 3. 阈值逻辑验证

程序中的阈值规则如下：

| 降雨强度 | 告警等级 | 系统行为 |
| ---: | --- | --- |
| < 10 mm/h | 绿色 | 正常显示，不写入告警日志 |
| 10 至 < 20 mm/h | 黄色 | 显示关注提示，不写入红色告警日志 |
| >= 20 mm/h | 红色 | 显示强降雨告警，并写入 `alert_log.txt` |

已通过边界值断言验证：

- 0.0 mm/h -> 绿色
- 9.9 mm/h -> 绿色
- 10.0 mm/h -> 黄色
- 19.9 mm/h -> 黄色
- 20.0 mm/h -> 红色

## 4. 红色告警日志验证

使用示例数据 `21.0 mm/h` 测试红色告警，系统已写入 `alert_log.txt`。

日志内容包含：

- 时间戳
- 城市名称
- 降雨强度
- 告警等级
- 数据来源

结论：红色告警触发与日志记录功能正常。

## 5. 物理合理性验证

OpenWeatherMap 当前天气接口返回的是近实时天气数据。如果返回结果中没有 `rain` 字段，程序将降雨强度解释为 `0.0 mm/h`，这与当前无降雨或云量较多但未降水的天气场景一致。

根据实验阈值：

- 当前北京 `0.0 mm/h` 小于 `10 mm/h`，绿色正常。
- 当前广州 `0.3 mm/h` 虽显示小雨，但远低于黄色阈值，绿色正常。
- 当示例数据设置为 `21.0 mm/h` 时，已超过 `20 mm/h` 强降雨告警阈值，红色告警合理。

## 6. AI 错误与修正

- 原始英文文档中的阈值符号在提取时曾出现编码显示异常，已人工确认并按 `< 10`、`10 至 < 20`、`>= 20` 实现。
- 无 API Key 时无法展示真实天气，因此增加了“示例数据”开关，用于课堂演示和阈值测试。
- 示例数据和真实 API 数据混合显示会影响历史曲线理解，因此已将 `rainfall_history.csv` 清理为真实 API 观测记录。
