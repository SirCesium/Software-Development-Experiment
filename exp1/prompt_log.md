## 交互 2：生成 API 调用代码

**中文提示词：**

> 我是水资源专业学生，请帮我写 Python 代码调用 OpenWeatherMap 当前天气接口，获取北京的降雨强度。代码需要使用 requests，能够提取 rain.1h 或 rain.3h，并对接口错误给出清晰中文提示。

**AI 输出摘要：**

- 使用 `requests.get()` 调用 `https://api.openweathermap.org/data/2.5/weather`。
- 参数包含城市名、API Key、摄氏温度单位和中文天气描述。
- 如果返回 `rain.1h`，直接作为小时降雨强度。
- 如果只有 `rain.3h`，除以 3 近似换算为 mm/h。
- 无降雨字段时返回 0 mm/h。

**人工校正：**

- 将接口错误统一封装为 `WeatherAPIError`，便于仪表盘显示中文错误信息。

## 交互 3：实现告警逻辑

**中文提示词：**

> 请按实验要求实现降雨阈值告警：小于 10 mm/h 为绿色，10 到 20 mm/h 为黄色，大于等于 20 mm/h 为红色。红色告警需要写入日志，日志必须包含时间、城市、降雨强度和告警等级。

**AI 输出摘要：**

- `check_alert()` 返回结构化的告警等级、颜色、中文提示和是否写日志。
- `log_alert()` 仅在红色告警时追加写入 `alert_log.txt`。
- 日志时间格式使用 `YYYY-MM-DD HH:MM:SS`。

**人工校正：**

- 增加了命令行自检，验证 9.9、10、19.9、20 等边界值。

## 交互 4：构建 Streamlit 仪表盘

**中文提示词：**

> 请构建 Streamlit 仪表盘，标题为 Rainfall Monitor - 城市名，显示当前降雨强度、彩色告警状态、天气描述、数据来源、更新时间和历史降雨曲线。页面需要每 5 分钟自动刷新。

**AI 输出摘要：**

- 使用 `st.metric()` 展示当前降雨强度。
- 使用彩色状态块显示绿色、黄色或红色告警。
- 使用 `st.line_chart()` 显示最近历史记录。
- 使用 HTML meta refresh 实现 300 秒自动刷新。

**人工校正：**

- 侧边栏保留“使用示例数据”开关和示例降雨强度滑块，方便无网络或无 API Key 时完成演示。

## AI 错误与修正记录

- 原始实验文档为英文，AI 初始提取时存在部分字符编码显示异常，例如 `≥` 被显示为乱码；实现时已根据上下文恢复为正确阈值含义。
- 若没有 API Key，直接调用真实接口会导致程序不可演示；已增加示例数据模式作为修正。
- Streamlit 自动刷新没有额外引入第三方组件，改用内置 Markdown 注入 meta refresh，减少依赖。

## 物理合理性说明

- 0 mm/h 代表当前无降雨，在晴天或非降雨天气中合理。
- 10 mm/h 已经属于需要关注的中等到较强降雨，可能造成低洼点积水。
- 20 mm/h 达到本实验设置的强降雨告警阈值，应触发城市排水和巡查响应。
- 若真实接口返回短时间内极高降雨值，需要结合当地雷达、雨量站或气象台记录进一步验证。
