## 2. 已知样例验证

文档给出的样例为：

- `P = 50 mm`
- `CN = 80`
- `S = (25400 / 80) - 254 = 63.5 mm`
- `Ia = 0.2 * S = 12.7 mm`

程序计算：

```text
Q = 13.8 mm
```

与实验文档期望结果一致，并满足 `13.8 <= 50`。

## 3. 敏感性分析结果

固定 `P = 50 mm` 时，不同 CN 的径流结果如下：

| CN | Q (mm) | Q/P |
| ---: | ---: | ---: |
| 60 | 1.40 | 0.028 |
| 70 | 5.81 | 0.116 |
| 80 | 13.80 | 0.276 |
| 90 | 27.11 | 0.542 |
| 95 | 36.90 | 0.738 |
| 100 | 50.00 | 1.000 |

结论：

- CN 越高，地表越不透水或入渗能力越弱，因此直接径流越大。
- `CN = 60` 类似良好植被覆盖，径流很小。
- `CN = 100` 代表完全不透水地表，全部降雨转化为径流。

## 4. 物理合理性说明

SCS-CN 方法中，CN 同时反映土地利用、土壤类型和入渗能力。

- 较低 CN：滞蓄量 `S` 大，初损 `Ia` 大，需要更大降雨才会形成径流。
- 较高 CN：滞蓄量 `S` 小，初损 `Ia` 小，更容易形成径流。
- 当 `P <= Ia` 时，降雨主要被截留、洼蓄或入渗，因此 `Q = 0` 合理。
- 程序通过裁剪保证 `Q <= P`，符合水量守恒约束。

## 5. AI 错误与修正

- AI 初始实现容易忽略 `CN = 0` 的除零问题，最终使用 `CN = 0 -> Q = 0` 作为物理边界。
- AI 可能只处理标量输入，最终增加 NumPy 向量化支持，便于敏感性分析。
- `P = Ia` 边界需要返回 0，最终使用 `effective_rainfall > 0` 判断，避免边界误差。
- 图中增加 `Q = P` 参考线，用于直观看出所有曲线都不超过降雨上限。
